# Afisha
В этом проекте мы будем работать с реальными данными сервиса Яндекс Афиша. Он позволяет пользователям узнавать о мероприятиях в разных городах и покупать на них билеты. Яндекс Афиша сотрудничает с организаторами мероприятий и билетными операторами, которые публикуют информацию и выставляют билеты на продажу.

***Цель***

Не просто привлекать новых пользователей, а удерживать их и превращать в постоянных. Для этого нужно повысить долю клиентов, которые после одной покупки приходят снова и совершают новые.
Нам нужно помочь команде маркетинга лучше понять поведение пользователей. Для этого проведем исследовательский анализ данных, чтобы выяснить, какие пользователи с большей вероятностью возвращаются на платформу и делают заказы повторно. Это позволит:
 - Быстро выявлять перспективных клиентов и предлагать им персонализированные условия.
 - Точно настраивать рекламу на аудитории с высокой вероятностью возврата.
 - Оптимизировать маркетинговые бюджеты.
 - Повысить общий уровень удержания клиентов.

***Структура проекта***

Проект состоит из пяти частей, которые последовательно проведут нас от подключения к данным до финальной публикации работы.

1. Знакомство с данными. 

Подключимся к базе данных SQL и проведём первичное знакомство с данными.

2. Подготовка к выгрузке данных.

Подготовим запрос для выгрузки данных, которые потом будем анализировать инструментами Python.

3. Исследовательский анализ данных.

На этом этапе пройдём несколько шагов:

  - Выгрузим данные из БД SQL в датафрейм pandas, чтобы выполнять дальнейшую работу с помощью Python и библиотек анализа данных.
  - Выполним предобработку данных.
  - Сформируем профиль пользователя, выделив признаки и факторы, которые могут влиять на повторные заказы.
  - Выясним, как распределены данные и какие закономерности в них наблюдаются.
  - Проведём корреляционный анализ и вычислим коэффициенты phik, чтобы узнать, какие признаки влияют на количество заказов.

4. Выводы.

Опираясь на результаты исследовательского анализа данных, сформулируем рекомендации для бизнеса.

5. Публикация проекта в Git.
 
Подготовим проект для публикации его в репозитории Git.
